## Project overview

> First I build the semantic analyzer by follwing the spec( entry table, symbol table and symbol manager), the entry holds symbol name, kind, level, type and its attributes. symbol manager handle all actions about the table by their scope, then visit all the node to construct the AST and therefore bulid the symbol table.

> Semantic analysis is performed at each node, conducting semantic checks after traversing all its child nodes. If an error is detected in the current node, it returns to its caller and indicates that it is faulty. Faulty child nodes frequently occur in expression nodes.

> To summarize, semantic checking is performed during the construction of the AST in a bottom-up manner using a symbol manager and a semantic analyzer. When an error is detected, an error message is generated by calling the ErrorPrinter. After completing the traversal, the error messages are outputted.
